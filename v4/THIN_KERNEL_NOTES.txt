Task-family kernel notes
=======================

Main changes relative to v3
---------------------------
1. Removed the generic ResidualRuntimeSession class.
2. Removed the generic workflow step dispatcher over kind=host/role/foreach.
3. Removed the generic resolve_expr()/resolve_path() interpreter for workflow bindings.
4. Emitted dedicated role runners:
   - run_planner_role(...)
   - run_implementer_role(...)
5. Emitted a dedicated program entrypoint:
   - run_autoresearch_program(...)
   - aliased as run_workflow(...)

What remains generic on purpose
-------------------------------
- KappaClient
- ToolHost
- parser / validator library
- one shared phase executor (_execute_phase)

Kernel size comparison
----------------------
v3 compiled_kernel.py: 887 lines
v4 compiled_kernel.py: 459 lines

Interpretation
--------------
The kernel is now thinner in the Futamura sense:
- the workflow graph is no longer interpreted at runtime;
- role dispatch is no longer interpreted by name;
- host-op dispatch is no longer interpreted by registry lookup.

The remaining generic core is now mostly the κ-boundary executor for a single phase.
